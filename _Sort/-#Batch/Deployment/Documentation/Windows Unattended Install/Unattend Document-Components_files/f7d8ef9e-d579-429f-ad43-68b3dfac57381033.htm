BitsServerExtensionsISAPI

Specifies whether to install Internet Server Application Programming Interface (ISAPI) for Background Intelligent Transfer Service (BITS) server extensions on client computers.

BITS provides two very useful features to the client:

•

Automatically adjusts the amount of network bandwidth used to make sure all other foreground applications use up all the bandwidth they need, then BITS uses up the rest.

•

Automatically continues a file transfer if the download (or upload) was interrupted by a computer restart or an unplugged network cable.

Syntax
        BitsServerExtensionsISAPI = On | Off
ValueDescription

On

Installs ISAPI for BITS server extensions on client computers.

Off

Does not install ISAPI for BITS server extensions on client computers.

Example
[Components]
BitsServerExtensionsISAPI = Off
Comments

The default value is Off. This entry requires BITSServerExtensionsManager = On and IIS_WWW = On in the [Components] section.

IEAccess

Specifies whether to install visible entry points to Internet Explorer.

Syntax
        IEAccess = On | Off
ValueDescription

On

Installs visible entry points to Internet Explorer.

Off

Does not install visible entry points to Internet Explorer.

Example
[Components]
IEAccess = On
Comments

The default value is On for Windows XP. The IEAccess entry is not available for Windows Server 2003. This entry does not remove any Windows code, including any Internet Explorer code, other than the visible entry points. Various Windows components and non-Microsoft applications rely upon Internet Explorer code.

IEAccess has no effect on Internet Explorer branding or the configuration of Internet settings.

You must use the entry IEAccess = Off and then install a different Internet Explorer installation package on the computer, so that the user can see the Internet Explorer visible entry points. Other forms of subsequent Internet Explorer configuration (such as autoconfiguration of Internet settings) do not affect whether the Internet Explorer visible entry points are visible on the Windows desktop.

IIS_Common

Specifies whether to install the common set of files required by IIS.

Syntax
        IIS_Common = On | Off
ValueDescription

On

Installs the common set of files required by IIS.

Off

Does not install the common set of files required by IIS.

Example
[Components]
IIS_Common = On
Comments

The default value is On for Windows Server 2003, Web Edition and Microsoft Small Business Server. The default value is Off for Windows XP, Windows Server 2003, Standard Edition, Windows Server 2003, Enterprise Edition, and Windows Server 2003, Datacenter Edition.

The following components require the IIS_Common entry:

•

All of the IIS components (IIS_FTP, IIS_InetMgr, IIS_NNTP, IIS_SMTP, and IIS_WWW)

•

FrontPage Extensions (FP_Extensions)

•

Indexing Service (IndexSrv_System)

•

Message Queuing (MSMQ_Core)

•

Windows Media Server Web-based administration tools (WMS_Admin_ASP)

•

Microsoft Exchange Server

•

Microsoft BackOffice Server

Any component that requires the IIS component will install it even if IIS_Common = Off.

Note

•

Windows XP Professional contains Internet Information Services 5.1. All editions of Windows Server 2003 contain Internet Information Services 6.0.

•

Windows XP Home Edition does not contain any Web server software components.
